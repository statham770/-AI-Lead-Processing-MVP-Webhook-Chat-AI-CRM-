# 🎯 AI Lead Processing MVP (Webhook + Chat → AI → CRM)

Автоматизована система інкубації, кваліфікації та маршрутизації лідів на базі **n8n**, локального штучного інтелекту (**Ollama**) та інтеграції з **Google Sheets** і **Telegram**.

Даний репозиторій містить готовий конфігураційний файл воркфлоу n8n, інструкцію з його розгортання та демонстраційне відео роботи системи.

---

## 🏗️ 1. Опис логіки та архітектури воркфлоу

Система спроєктована за модульним принципом для забезпечення максимальної стабільності обробки даних, незалежно від джерела надходження запиту. Вона об'єднує два незалежних канали лідогенерації в єдиний аналітичний хаб.

### 🔄 Покрокова схема руху даних:

```
[ Webhook Trigger ] ──┐
                      ├──> [ Data Normalization ] ──> [ AI Processing (Ollama) ] ──> [ Parse LLM Response ]
[ Chat Trigger ] ─────┘                                                                        │
                                                                                               ▼
[ Chat Response ] <── [ Telegram Node ] <── [ Format Telegram Message ] <── [ Google Sheets Node ]
```

### 📋 Детальний розбір роботи нод:

| № | Назва ноди | Тип / Рушій | Опис бізнес-логіки та функцій |
|---|---|---|---|
| **1a** | `Webhook` | Webhook Trigger | Приймає сирі `POST` запити від зовнішніх джерел (лендінги, лід-форми, сторонні CRM, чат-боти). |
| **1b** | `Chat Trigger` | LangChain Chat Trigger | Інтерфейсна нода вбудованого чату n8n, що дозволяє взаємодіяти з AI безпосередньо через веб-панель. |
| **2** | `Data Normalization` | Code (JavaScript) | **Критичний вузол.** Приводить дані з обох тригерів до єдиного об'єкта `normalized_payload`. Це ізолює подальшу логіку від змін у структурі вхідних запитів. |
| **3** | `AI Processing` | LangChain Chain | Надсилає нормалізований текст запиту до мовної моделі за допомогою структурованого системного промпту. |
| **4** | `Ollama Model` | Chat Ollama | Локальний ШІ-рушій. Використовує модель (наприклад, `qwen3.5:9b` або `gemma`), працює з низькою температурою (`0.1`–`0.2`) для мінімізації галюцинацій та точного виокремлення сутностей. |
| **5** | `Parse LLM Response` | Code (JavaScript) | **Логічний центр очищення.** За допомогою регексів витягує чистий JSON з тексту AI (навіть якщо модель додала маркдаун). Відновлює оригінальний текст запиту, запобігаючи помилкам `undefined`, та автоматично проставляє таймстампи. |
| **6** | `Google Sheets` | Google Sheets | Виконує роль CRM/Бази даних. Автоматично мапить очищені ШІ поля у відповідні колонки таблиці (Ім'я, Компанія, Бюджет, Температура ліда тощо). |
| **7** | `Format Telegram Message` | Code (JavaScript) | Формує структурований, емодзі-варіований звіт для команди продажів (задає жирні шрифти, італік, правильні відступи). |
| **8** | `Telegram` | Telegram Node | Надсилає згенерований звіт у реальному часі у ваш внутрішній Telegram-канал або робочу групу. |
| **9** | `Chat Response` | Code (JavaScript) | Повертає в інтерфейс `Chat Trigger` фінальну чисту текстову строку (*"Дякуємо! Ваша заявка прийнята..."*), повністю приховуючи технічні логі від користувача. |

## 🛠️ 2. Специфікація ШІ-аналізу (Промптинг та Парсинг)

Для кваліфікації ліда модель аналізує текст та виділяє 6 ключових параметрів:
* **`summary`** — Короткий зміст запиту українською мовою.
* **`lead_temperature`** — Оцінка готовності клієнта до покупки: `Hot` (купити вже зараз), `Warm` (проявляє інтерес, порівнює), `Cold` (просто запитання).
* **`type`** — Сегмент ринку: `B2B` (бізнес для бізнесу) або `B2C` (робота з приватними клієнтами).
* **`company`** — Назва організації (якщо згадана).
* **`phone`** — Контактний номер телефону.
* **`budget`** — Фінансові рамки запиту (наприклад, *"3000 доларів"*).

Завдяки скрипту в `Parse LLM Response`, система захищена від збоїв: якщо локальна модель не знаходить телефон або бюджет, вона автоматично присвоює значення `"Не вказано"`, не ламаючи структуру бази даних.

## 🚀 3. Інструкція із запуску та розгортання

### 📋 Попередні вимоги (Prerequisites)
Перед імпортом воркфлоу переконайтеся, що у вас розгорнуті та налаштовані такі компоненти:
1.  **n8n** (локально через Docker або в хмарі n8n Cloud).
2.  **Ollama** (встановлена на тому ж сервері/ПК, де працює n8n, або доступна по мережі).
3.  Завантажена модель в Ollama (рекомендовано `qwen3.5:9b`, `gemma2` або `llama3`). Команда для завантаження:
    ```bash
    ollama run qwen3.5:9b
    ```
4.  **Telegram-бот**, створений через [@BotFather](https://t.me/BotFather) (знадобиться *Bot Token* та *Chat ID* цільового каналу).
5.  **Google-таблиця** з налаштованим доступом через Google Cloud Console (OAuth2 або Service Account).

---

### ⏱️ Покрокове налаштування воркфлоу

#### Крок 1. Імпорт сценарію в n8n
1.  Створіть новий порожній воркфлоу в n8n.
2.  Натисніть на меню (три крапки у верхньому правому куті) -> **Import from File**.
3.  Виберіть конфігураційний файл `Lead Processing MVP (Webhook + Chat -> AI -> CRM).json` з цього репозиторію.

#### Крок 2. Конфігурація Ollama
1.  Відкрийте круглу ноду **Ollama Model**.
2.  В полі **Base URL** вкажіть адресу вашого локального сервера Ollama (зазвичай `http://localhost:11434` або `http://host.docker.internal:11434` для Docker).
3.  В полі **Model Name** впишіть назву завантаженої моделі (наприклад, `qwen3.5:9b`).
4.  *Важливо:* Переконайтеся, що параметр `Output Format: JSON` вимкнено (видалено через іконку кошика), оскільки парсинг повністю забезпечується наступною JavaScript-нодою.

#### Крок 3. Підготовка Google Sheets
1.  Створіть нову Google-таблицю.
2.  У **першому рядку (Header Row)** створіть стовпчики з такими точними назвами (регістр має значення):
    `leadSummary` | `leadTemperature` | `leadCategory` | `companyName` | `phoneNumber` | `budget` | `inquiryMessage` | `processedTime`
3.  В n8n відкрийте ноду **Google Sheets**, оберіть ваш Google-аккаунт, вкажіть `Spreadsheet ID` та виберіть потрібний лист (`Sheet`). Режим роботи повинен бути встановлений на **Append row to sheet**.

#### Крок 4. Налаштування Telegram сповіщень
1.  Відкрийте ноду **Telegram**.
2.  Оберіть або створіть нове підключення (Credentials), вставивши токен вашого бота.
3.  В полі **Chat ID** вкажіть ідентифікатор вашої групи або каналу (наприклад, `-100XXXXXXXXXX`).
4.  Переконайтеся, що бот доданий у цей канал як адміністратор з правами відправки повідомлень.

#### Крок 5. Активація
Натисніть кнопку **Save** у верхньому правому куті та переведіть перемикач воркфлоу у стан **Active**.

## 🛠️ 5. Можливі проблеми та їх вирішення (Troubleshooting)

* **Проблема:** У Telegram приходить повідомлення зі значенням `Inquiry: undefined`.
    * *Рішення:* Переконайтеся, що ви оновили код у ноді `Parse LLM Response` на версію з відновленням оригінального тексту через змінну `originalMessage`. Нода повинна повертати ключ `inquiryMessage`.
* **Проблема:** Чат n8n виводить відповідь у вигляді JSON-коду з дужками `{}`.
    * *Рішення:* Перевірте ноду `Chat Response`. Фінальний об'єкт обов'язково повинен повертати ключ з назвою `text` (а не `response`), оскільки інтерфейс чату n8n реагує виключно на маркер `text` для показу чистого повідомлення.
* **Проблема:** Нода AI Processing видає помилку `Unexpected end of JSON input`.
    * *Рішення:* Локальна модель намагається повернути пояснювальний текст разом із JSON, або в ноді Ollama Model активовано примусовий формат JSON, який обриває генерацію. Вимкніть параметр `Format: JSON` в Ollama Model та знизьте температуру до `0.1`.
